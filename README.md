# Кулинарная книга

[![CI](https://github.com/VladKoretski/cooking_book/actions/workflows/ci.yml/badge.svg)](https://github.com/VladKoretski/cooking_book/actions/workflows/ci.yml)

Программа для чтения рецептов из текстового файла и расчёта количества ингредиентов на заданное число персон.

## Описание

Проект написан в рамках домашнего задания к лекции «Открытие и чтение файла, запись в файл».  
Код полностью соответствует условию: читает файл с рецептами, формирует словарь `cook_book` и предоставляет функцию для составления списка покупок.


## Требования

- Python 3.8 или выше
- Стандартная библиотека (дополнительные пакеты не требуются)

## Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/ваш_username/cooking-book.git
cd cooking-book
```

(Опционально) Создайте виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate  # для Linux/Mac
venv\Scripts\activate     # для Windows
```

## Использование

1. Поместите файл с рецептами в папку `data/recipes.txt` или измените путь в коде (переменная `path_to_file`).
2. Запустите скрипт:

```bash
python cooking.py
```

В коде уже есть пример вызова:

```python
cook_book = file_into_dictionary_recipes(path_to_file)
print(get_shop_list_by_dishes(['Запеченный картофель', 'Омлет', 'Утка по-пекински', 'Колбаса'], 3))
```

Вы можете импортировать функции в своих скриптах:

```python
from cooking import file_into_dictionary_recipes, get_shop_list_by_dishes

cook_book = file_into_dictionary_recipes('recipes.txt')
shopping_list = get_shop_list_by_dishes(['Омлет'], 2)
print(shopping_list)
```

## Формат файла рецептов

Файл должен иметь следующую структуру (пустые строки игнорируются):

```
Название блюда
Количество ингредиентов
Название ингредиента | Количество | Единица измерения
Название ингредиента | Количество | Единица измерения
...
```

Пример (файл `recipes.txt`):

```
Омлет
3
Яйцо | 2 | шт
Молоко | 100 | мл
Помидор | 2 | шт

Утка по-пекински
4
Утка | 1 | шт
Вода | 2 | л
Мед | 3 | ст.л
Соевый соус | 60 | мл
```

**Важно:** название блюда не должно содержать символ `|` и не должно быть числом (это особенность текущей реализации).

## Функции

### `line_parcing(line_with_ingrdients)`

Парсит строку с ингредиентом.

- **Аргументы:** `line_with_ingrdients` (str) – строка вида `"Название | Количество | Единица"`.
- **Возвращает:** словарь с ключами `ingredient_name`, `quantity`, `measure` или `True`, если строка не содержит `|` (используется для определения названия блюда).

### `file_into_dictionary_recipes(recipes_file)`

Читает файл и создаёт словарь `cook_book`.

- **Аргументы:** `recipes_file` (str) – путь к файлу.
- **Возвращает:** словарь, где ключ – название блюда, значение – список ингредиентов (каждый ингредиент – словарь).

### `get_shop_list_by_dishes(dishes, person_count)`

Рассчитывает общее количество ингредиентов для указанных блюд.

- **Аргументы:** 
  - `dishes` (list) – список названий блюд.
  - `person_count` (int) – количество персон.
- **Возвращает:** словарь с ингредиентами и их общим количеством.
- **Примечание:** использует глобальную переменную `cook_book`, которая должна быть предварительно заполнена.

## Тестирование

Для запуска тестов выполните:

```bash
python -m unittest discover tests
```

Тесты проверяют:

- Парсинг строки ингредиента.
- Чтение файла и формирование `cook_book`.
- Расчёт списка покупок (включая обработку отсутствующих блюд).

## Непрерывная интеграция (CI)

При каждом пуше в ветку `main` или создании pull request автоматически запускаются:

- Линтер `flake8` (проверка стиля кода).
- Все модульные тесты.

Конфигурация находится в `.github/workflows/ci.yml`.
